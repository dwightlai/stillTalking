# Still Talking 持续发布与分发手册

## 发布节奏

- 每周发布 1–2 篇，不为追求数量降低 Rule Zero。
- 周二优先发布指导型或问答型文章；周五优先发布叙事型或研究解读型文章。
- 首次上线使用真实 `publishedAt`，实质修改使用 `updatedAt`。
- 选题进入写作前，先确认一个可追溯的真实元素和一个与其他文章不同的结构。

## 未来 20 篇形式配比

| 形式 | 目标篇数 | 使用条件 |
|---|---:|---|
| 指导型 | 8 | 有具体场景、行为和可执行动作 |
| 问答型 | 5 | 只使用获得许可的真实来信 |
| 叙事型 | 4 | 真实投稿或明确标注的复合场景 |
| 研究解读型 | 3 | 以原始研究或政府数据为核心 |

没有真实来信时，问答型名额顺延，不伪造投稿。

## 单篇发布流程

1. 写一页选题简报：读者问题、独特论点、真实元素、来源、文章形式。
2. 完成正文后运行 `npm run content:check`。
3. 人工检查对话是否像美国人会自然说出口。
4. 核对研究样本、年份、相关性和因果边界。
5. 为文章生成独立插图并检查桌面、手机裁切。
6. 运行 `npm run lint` 和 `npm run build`。
7. 发布后制作分发材料，不在正文完成前倒推标题。

## 每篇文章的分发包

- 30–60 秒短视频脚本：一个场景、一个转折、一个可执行动作。
- Pinterest：一张独立竖版图和 2–3 句摘要。
- Quora：先完整回答问题，再在确有帮助时链接原文。
- Reddit：只以真实身份和真实参与记录发言，不伪装读者，不批量投放链接。
- 所有可控外链添加 `utm_source`、`utm_medium` 和 `utm_campaign`。

## 月度复盘

- Umami：访问来源、热门文章、回访和文章间流动。
- Clarity：阅读中断位置、移动端误触和页面滚动。
- Kit：新增订阅、确认率、来源页面、退订和点击。
- 内容：哪种真实元素带来更长停留，而不是只比较标题点击率。
