# Still Talking 内容改进行动计划

依据：`Still_Talking_网站内容分析与改进建议.md`

## 执行原则

1. 真实细节优先：每篇文章必须有可感知、可追溯的人物行为、生活场景、
   自然对话、真实经历或权威研究，让读者记住一个具体画面，而不只是要点。
2. 不虚构作者身份、职业资历、读者来信或个人经历。
3. 不为制造“持续更新”假象而回填虚假发布日期。
4. 研究数据只引用可核查的原始研究、政府数据或权威调查。
5. 案例若非真实投稿，必须明确标注为复合场景。
6. 文章首先服务读者；SEO、订阅和商业链接不能损害可信度。

## 第一阶段：信任基础

- [x] Kit 邮件订阅接入生产环境。
- [x] 在 About 页面说明采用匿名团队署名及其编辑边界。
- [x] About 页面增加“为什么建立 Still Talking”和内容如何形成。
- [x] 保留 `Still Talking Editors` 团队署名，不创建虚构个人作者。
- [x] 使用编辑场景插画，不使用冒充真人的 AI 头像。

验收标准：

- 网站不暗示不存在的临床资质、采访或真人案例。
- About 页面明确网站立场、动机和证据标准。

## 第二阶段：证据与真实元素

- [x] 为 12 篇现有文章各补充至少一个可核查来源。
- [x] 在相关论点附近自然说明研究或数据的适用范围。
- [x] 文章页统一展示 Sources & further reading。
- [x] 复合场景统一披露，不再让类型化案例看似真人采访。
- [x] 内容检查脚本强制校验内容形式、更新时间及来源。

验收标准：

- 每篇文章至少包含一个权威来源或经确认的真实元素。
- 所有数字和非显而易见的事实均可追溯。
- 内容构建会拒绝缺作者、缺来源或未来发布日期的文章。

## 第三阶段：内容形式与发布节奏

- [ ] 新文章按每周 1–2 篇发布，不批量灌入。
- [x] 保留真实首次发布日期，修改后单独记录 `updatedAt`。
- [ ] 未来 20 篇目标配比：指导型 40%、问答型 25%、叙事型 20%、研究解读型 15%。
- [ ] 问答型内容只使用获得许可的真实来信；没有来信时不伪造。
- [ ] 叙事型内容使用真实投稿或明确披露的复合场景。

验收标准：

- 不出现未来发布日期。
- 连续文章不重复相同开头、章节顺序和结尾动作。
- Library 继续按真实发布日期倒序。

## 第四阶段：邮件与分发

- [x] Kit 表单、双重确认和来源页面记录已接入。
- [ ] 在 Kit 中完成欢迎邮件及 Still Talking Weekly 模板。
- [ ] 每篇新文章制作一个短视频脚本、一个 Pinterest 文案和一个问答摘要。
- [ ] 社交发布由真人账号完成，不使用批量自动灌水。
- [ ] 每月复盘来源、订阅转化、确认率和高停留文章。

验收标准：

- 新订阅者完成确认后收到欢迎邮件。
- 每个分发链接使用 UTM 参数。
- 不在 Reddit、Quora 等社区伪装成无关用户推广。

## 当前身份策略

站点不公开个人身份，因此采用透明的匿名编辑团队模式。不得用虚构姓名、
虚构家庭经历、AI 人像或不存在的资历来制造“真人感”。信任由可核查来源、
清晰案例标注、稳定编辑标准和持续发布记录建立。
